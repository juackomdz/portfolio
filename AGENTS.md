# AGENTS.md — Nuxt 4 Portfolio

## Commands

```bash
npm run dev          # dev server (port 3000)
npm run build        # production build
npm run generate     # static export (nuxt generate)
npm run preview      # preview production build
npm run postinstall  # required after dep changes (runs: nuxt prepare)
```

## Key Dependencies

- `nuxt` v4, `vue` 3.5, `vue-router` 5.1
- `tailwindcss` v4 via `@tailwindcss/vite` plugin in `nuxt.config.ts:45-47`
- `@nuxtjs/color-mode` — class-based dark mode (`portfolio-color-mode` storage key, class suffix `""`)
- `motion-v/nuxt` — Vue Motion animations
- `@vueuse/nuxt` — composables auto-imported
- `@nuxt/icon` — local SVG icon server bundle (`scan: true`, `serverBundle: 'local'`)
- `reka-ui` — headless UI primitives
- `@nuxt/fonts` — font optimization
- `tailwind-merge` — class merging utility
- `tailwind-variants` — `tv` and `VariantProps` globally auto-imported via `nuxt.config.ts:15-24`

## Tailwind v4

- Entry: `app/assets/css/tailwind.css` (imported via `css: ['~/assets/css/tailwind.css']`)
- `tw-animate-css` plugin imported in CSS
- `@tailwindcss/forms` with `strategy: "class"` — form inputs require `form-*` classes
- Custom OKLCH color tokens, animations (shine, gradientFlow, meteor, grid) defined in CSS
- `tw` helper in `app/utils/tw-helper.ts` for `tw` tagged template (configured in `.vscode/settings.json` for intellisense)

## Component Generator (ui-thing)

- Config: `ui-thing.config.ts` — outputs to `app/components/Ui/`, reads theme from `app/assets/css/tailwind.css`
- Run: `npx ui-thing add <component>`

## Architecture

- Single-page portfolio — all components composed in `app/app.vue` (no pages/ directory)
- Generated UI components live in `app/components/Ui/` (Card/\*, Badge, BorderBeam, Container, Separator, BgPattern/Square)
- App-specific components at `app/components/` (Background, Footer, Hero, Terminal, etc.)
- No `composables/` or `plugins/` exist yet

## TypeScript

- `tsconfig.json` references `.nuxt/tsconfig.*.json` (generated by `nuxt prepare`)
- No explicit `compilerOptions` — inherits from Nuxt

## Gotchas

- Always run `npm run postinstall` after adding/removing deps to regenerate `.nuxt/` types
- Dark mode uses `.dark` class on `<html>` (not media query); `@custom-variant dark` in CSS
- No CI, no ESLint/Prettier config, no test framework set up
