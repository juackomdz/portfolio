# AGENTS.md — Nuxt 4 Portfolio

## Project Overview
Minimal Nuxt 4 portfolio starter with Tailwind CSS v4 (via `@tailwindcss/vite`), dark mode, animations, and component generator (`ui-thing`).

## Critical Commands
```bash
# Dev server (port 3000 default)
npm run dev

# Build for production
npm run build

# Preview production build
npm run preview

# Type check (via nuxt/tsc)
npm run typecheck

# Post-install hook (required after dependency changes)
npm run postinstall   # runs: nuxt prepare
```

## Architecture & Conventions

### Directory Structure (under `/app`)
```
app/
├── app.vue              # Root layout: Background + Footer only
├── assets/css/tailwind.css   # Tailwind v4 config with custom theme, animations, OKLCH colors
├── components/          # App components (Background.vue, Footer.vue)
├── components/Ui/       # Generated UI components (via ui-thing)
├── composables/         # Auto-imported composables
├── plugins/             # Nuxt plugins
└── utils/               # Helper utilities (tw-helper.ts exports `tw` tagged template)
```

### Tailwind CSS v4 Setup
- **Plugin**: `@tailwindcss/vite` in `nuxt.config.ts:46`
- **Entry**: `app/assets/css/tailwind.css` (imported via `css: ['~/assets/css/tailwind.css']`)
- **Features**: Custom OKLCH color palette, custom animations (shine, gradientFlow, meteor, grid), dark mode via `.dark` class, `@tailwindcss/forms` with `strategy: "class"`
- **Utility**: `tw` helper in `app/utils/tw-helper.ts` for typed class strings

### Key Modules (nuxt.config.ts)
- `@nuxtjs/color-mode` — dark mode persistence (`portfolio-color-mode` key)
- `motion-v/nuxt` — Vue Motion animations
- `@vueuse/nuxt` — VueUse composables auto-imported
- `@nuxt/icon` — SVG icons (local server bundle, scans components)
- `@nuxt/fonts` — Font optimization
- `tailwind-variants` — `tv` and `VariantProps` globally imported

### Component Generator (ui-thing)
Config at `ui-thing.config.ts`:
- Outputs to `app/components/Ui/`
- Uses `app/assets/css/tailwind.css` for theme reference
- Package manager: npm
- Run via: `npx ui-thing add <component>` (or similar)

### TypeScript
- References `.nuxt/tsconfig.*.json` (generated by `nuxt prepare`)
- No explicit `compilerOptions` — inherits from Nuxt

## Agent Notes
- **Always run `npm run postinstall` after adding/removing deps** — generates `.nuxt/tsconfig.*` and types
- **Port 3000** is default; no custom port config found
- **No ESLint/Prettier configs** — use Nuxt defaults or add if needed
- **No test config found** — Vitest would be standard for Nuxt 4 but not set up yet
- **`tw` helper** is the pattern for Tailwind class composition in components
- **Dark mode** uses class strategy (`.dark` on `<html>`), not media query
